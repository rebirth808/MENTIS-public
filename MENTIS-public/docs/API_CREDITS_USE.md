# API Credits Use Plan

If MENTIS receives API credits, they would be used to test and improve real developer workflows involving Codex/OpenAI models.

## Primary goals

Credits would support:

- evaluating model routing for coding, reasoning, review, research, and testing tasks
- building reproducible Codex/OpenAI workflow examples
- generating and validating code changes across small open-source fixtures
- testing review loops with analyst, implementer, test engineer, critic, and integrator roles
- measuring quality, latency, and reliability across task types
- improving safety checks and human-in-the-loop execution
- writing public documentation and demos for other builders

## Example workflows

### Coding workflow

1. User describes a change.
2. MENTIS routes the task to a coding model/workflow.
3. A test engineer role proposes validation.
4. A critic role reviews risks and edge cases.
5. The human reviews before applying changes.

### Research workflow

1. User asks for a technical decision.
2. MENTIS gathers source-grounded context.
3. A reasoning role evaluates tradeoffs.
4. A critic checks assumptions.
5. The final answer includes uncertainty and decision notes.

### Safety workflow

1. A command or file operation is proposed.
2. Security gateway classifies risk.
3. Safe operations proceed.
4. Risky operations require confirmation.
5. Destructive or unsafe operations are blocked.

## What credits would not be used for

Credits would not be used for:

- spam
- fake benchmark inflation
- hidden automation without user control
- scraping private data
- credential extraction
- unsafe cyber activity
- non-project personal use
